# Mr.K Cover Studio PRD

产品域名：`dna.superk.ai`

## 1. 产品定位

Mr.K Cover Studio 是一个面向个人内容账号的「固定视觉系统封面生产器」。它把已经确认的 Mr.K 账号视觉资产固定下来，让用户只输入标题、栏目、英文副标题、摘要和背景图，就能快速生成统一风格的抖音主页背景、合集封面和单条作品封面。

一句话：把每次手工做封面的不确定性，变成一个可复用、可导出的账号视觉生产工具。

产品交付分为两种形态：Agent Skill 负责在对话里接收抖音链接并生成封面方案；Local Web Studio 负责可视化预览、调整、素材管理和导出。

## 2. 目标用户

第一阶段只服务一个明确用户：Mr.K 在路上，抖音号 KevPH2026。

后续可扩展为个人 IP 创作者、知识型短视频账号、AI/商业/成长类内容账号的封面模板工具。

## 3. 核心问题

当前封面制作存在三个问题：

1. 视觉系统容易跑偏：每次换背景、换标题后，字体、层级、头像、ID、K 标识容易不统一。
2. 输入内容没有结构：标题、栏目、英文标题、摘要、编号、口号等元素需要靠临时判断摆放。
3. 导出链路不稳定：主页背景、合集封面、单条作品封面尺寸不同，手工导出容易出错。

## 4. 产品目标

MVP 目标：

1. 固定 Mr.K 账号视觉系统。
2. 支持用户输入文字和上传/选择背景。
3. 支持根据文字主题 AI 生成配图，自动作为封面背景。
4. 一键生成三类图片：
   - 主页背景：1920x640
   - 合集封面：1080x1080
   - 单条作品封面：1080x1920
5. 支持导出 PNG。
6. 让用户在 1 分钟内完成一张可发布封面。

非目标：

1. 不做通用海报设计器。
2. 不开放自由拖拽所有元素。
3. 不做复杂图层编辑。
4. 不接抖音后台自动上传，第一版只负责本地生成。

## 7. 模板规格

### 7.1 主页背景

尺寸：1920x640

用途：抖音个人主页背景。

固定元素：

1. 右侧品牌文案：Deep thoughts on AI
2. 副文案：Business / Human Nature / Field Notes
3. KevPH2026
4. K 字标
5. 绿色识别线

可变元素：

1. 背景图
2. 是否显示文案版/干净版

安全规则：

1. 左侧 55% 尽量不放强文字，留给抖音头像、昵称和简介。
2. K 字标固定在右侧视觉区。
3. 文案不超过两行。

导出文件名：

`mrk-profile-background-{variant}.png`

### 7.2 合集封面

尺寸：1080x1080

用途：抖音合集封面。

固定元素：

1. 顶部品牌：MR.K 在路上
2. ID：KevPH2026 / Deep thoughts on AI
3. 头像印章
4. SERIES 标签
5. K 字标水印
6. 口号：少和人对话，多和AI对话

可变元素：

1. 栏目名称
2. 英文栏目名
3. 栏目副文案
4. 背景图
5. 栏目色

三类默认栏目：

1. AI下半场
   - 英文：AI SECOND HALF
   - 副文案：模型 / Agent / 商业重构
   - 颜色：荧光绿
2. 强者恒强
   - 英文：THE STRONG GET STRONGER
   - 副文案：判断力 / 筹码 / 个体系统
   - 颜色：青蓝
3. 在路上
   - 英文：ON THE ROAD
   - 副文案：出差 / 客户 / 一线观察
   - 颜色：红橙

导出文件名：

`collection-{category-slug}.png`

### 7.3 单条作品封面

尺寸：1080x1920

用途：短视频/图文作品封面。

固定元素：

1. 顶部品牌
2. ID
3. 头像印章
4. 栏目标签
5. 主标题区
6. 英文标题
7. 识别线
8. 摘要/点击理由
9. 口号
10. 作品编号
11. K 字标水印

可变元素：

1. 栏目
2. 中文标题
3. 英文标题
4. 摘要
5. 背景图
6. 作品编号

标题规则：

1. 中文标题优先最大。
2. 英文标题作为辅助，不抢主标题。
3. 摘要必须解释点击理由，不写泛泛口号。
4. 标题超过 8 个中文字符时自动换行。

导出文件名：

`work-{category-slug}-{code}.png`

## 9. 交互流程

### 9.1 创建单条作品封面

1. 用户进入工作台。
2. 选择「单条作品封面」。
3. 选择栏目：AI下半场 / 强者恒强 / 在路上。
4. 输入中文标题。
5. 输入英文标题。
6. 输入摘要。
7. 输入作品编号。
8. 选择或上传背景图。
9. 实时预览。
10. 点击导出 PNG。

### 9.2 创建合集封面

1. 选择「合集封面」。
2. 选择栏目。
3. 可调整英文栏目名和副文案。
4. 选择背景图。
5. 点击导出 PNG。

### 9.3 创建主页背景

1. 选择「主页背景」。
2. 选择背景图。
3. 选择安全区版或干净版。
4. 点击导出 PNG。

## 11. 技术方案建议

第一版建议做成本地应用：

1. HTML/CSS/JavaScript
2. Canvas 渲染
3. 本地图片上传，不传服务器
4. 一键导出 PNG
5. 所有资产放在本地目录
6. 本地 Python 服务提供 `/api/generate-image`
7. 服务端读取 `OPENAI_API_KEY` 调用 OpenAI Image API，浏览器端不保存 API Key
8. AI 接口不可用时，前端自动使用本地 Canvas 生成占位主题背景

理由：

1. 交付快。
2. 不需要登录和后端。
3. 图片隐私更安全。
4. API Key 不暴露在浏览器。
5. 适合本地创作流。

后续如果产品化给多人使用，再升级为 React/Vite + 服务端模板存储。

## 12. 验收标准

1. 用户可以在页面内输入标题、英文标题、摘要、编号。
2. 用户可以选择三类栏目并看到颜色变化。
3. 用户可以上传背景图并看到实时预览。
4. 用户可以根据标题/摘要/栏目一键请求 AI 生成主题配图背景。
5. AI 生成失败或未配置 Key 时，页面给出明确提示并生成本地占位背景。
6. 用户可以导出 1080x1920 单条作品封面。
7. 用户可以导出 1080x1080 合集封面。
8. 用户可以导出 1920x640 主页背景。
9. 导出的 PNG 与当前固定视觉系统一致。
10. 固定元素不能被误删或随意拖动。
11. 页面打开即是工作台。
12. 无需联网即可使用手动上传和本地占位生成。
